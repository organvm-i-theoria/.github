## Orchestration Rules

### Decision-Making Protocols

#### Rule 1: Default to "Not Yet"

When in doubt, keep something in personal incubation. Premature migration
creates pressure.

#### Rule 2: Single Organ Assignment

Nothing lives in two organs. If something seems to span purposes, it's not ready
to graduate.

#### Rule 3: Silence is Valid

Organs are allowed to be empty or quiet. Activity is not virtue.

#### Rule 4: Prohibitions Trump Permissions

If a repository violates an organ's prohibitions, it must move, even if it fits
permissions.

#### Rule 5: Orchestration Can Veto

Even if a repository fits an organ's criteria, orchestration can refuse
graduation if system integrity demands it.

### The Key Diagnostic Question

**Before adding anything to an organ, ask:**

> "Which existing boundary am I trying not to enforce?"

If the answer reveals avoidance, the urge is malformed. Do not act on it.

### Emergency Protocols

**If an organ becomes overwhelmed:**

1. **Stop intake immediately** - No new repos for 30 days
1. **Audit for violations** - Run boundary checks
1. **Enforce removals** - Move violating repos out
1. **Review prohibitions** - Ensure they're clear
1. **Resume cautiously** - One repo at a time

**If organs blur together:**

1. **Trace the contamination** - Which interface broke?
1. **Strengthen the boundary** - Update automation
1. **Document the lesson** - Add to decision log
1. **Don't add complexity** - Fix enforcement, don't expand definitions
